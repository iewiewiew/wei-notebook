现象：JMeter 非 GUI 模式压测卡住，无响应，服务端也没接收到请求  
原因：带宽太小限制住了

HTTP 压测过程中 MySQL 数据库可能存在什么问题？

性能瓶颈问题

1. **连接数耗尽**：大量并发请求导致数据库连接池被占满
   - 表现为"Too many connections"错误
   - 可能需要调整max_connections参数
2. **查询性能下降**：
   - 未优化的SQL查询在高并发下表现更差
   - 缺少适当索引导致全表扫描
   - 复杂JOIN操作成为瓶颈
3. **锁竞争加剧**：
   - 行锁、表锁等待时间增加
   - 死锁概率上升
   - 事务隔离级别影响并发性能
